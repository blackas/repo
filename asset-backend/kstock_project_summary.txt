[프로젝트 개요]
- 한국 주식 데이터를 수집(pykrx)하여 PostgreSQL에 저장
- 사용자별 관심종목을 기반으로 일일 리포트 생성
- Celery Beat 스케줄링으로 매일 리포트 생성 및 카카오 알림톡 발송

[구성 요소]
1) Django 프로젝트
   - config/settings.py: PostgreSQL, Celery, Kakao API 설정 포함
   - config/celery.py: Celery 앱 초기화
   - manage.py: Django 기본 실행 스크립트

2) 앱 구조
   - apps/accounts
     - User (전화번호, 알림 수신 여부)
     - WatchList / WatchListItem (관심종목 관리)
   - apps/stocks
     - Stock, DailyPrice 모델
     - services.py: pykrx 기반 종목 마스터/일별 시세 수집
     - management/commands/sync_korea_stocks.py: CLI로 데이터 동기화
   - apps/reports
     - DailyReport 모델
     - services.py: 관심종목 상승/하락 TOP3 기준 리포트 텍스트 생성
     - tasks.py: 모든 사용자에 대해 일일 리포트 생성 Celery 태스크
   - apps/notifications
     - NotificationLog 모델
     - services/kakao_client.py: 카카오 알림톡 API 래퍼 (senderKey + templateCode + recipientList payload)
     - tasks.py: DailyReport 를 카카오 알림톡으로 발송하는 Celery 태스크

3) 인프라
   - requirements.txt: Django, Celery, Redis, psycopg2, pykrx, requests 등
   - Dockerfile: Django/Celery를 위한 Python 3.11 slim 이미지
   - docker-compose.yml:
     - db(PostgreSQL), redis, web(Django), celery-worker, celery-beat 서비스
   - .env.example:
     - DB, Celery, Kakao API, Django 환경 변수 예시

[일일 처리 플로우 예시]
1. (선택) 오전 7:00 이전: `python manage.py sync_korea_stocks` 또는 동일 로직 Celery 태스크로 시세 동기화
2. 07:30: Celery Beat -> `create_daily_reports_for_all_users` 실행
3. 08:00: Celery Beat -> `send_daily_reports_via_kakao` 실행
4. NotificationLog 에 발송 결과/응답 코드 저장

이 zip을 풀고:
- `.env.example`를 `.env`로 복사 후 값 수정
- `docker-compose up -d` 실행
- `docker-compose exec web python manage.py migrate`
- `createsuperuser` 생성 후 admin에서 사용자/관심종목/리포트 상태 확인

하면 기본 기능을 바로 테스트할 수 있는 구조입니다.
